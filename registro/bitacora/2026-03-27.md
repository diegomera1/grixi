### [SECURITY] Aislamiento estricto de admin a grixi.grixi.ai
- **Archivos afectados:**
  - `app/lib/platform-guard.ts` — [NUEVO] Guard centralizado `isPlatformTenant()`
  - `app/routes/admin/index.tsx` — Guard en loader
  - `app/routes/admin/organizations.tsx` — Guard en loader + action
  - `app/routes/admin/organizations.$id.tsx` — Guard en loader + action
  - `app/routes/admin/users.tsx` — Guard en loader + action
  - `app/routes/admin/audit.tsx` — Guard en loader
  - `app/components/layout/sidebar.tsx` — Prop `tenantSlug`, admin solo si `tenantSlug === 'grixi'`
  - `app/routes/authenticated.tsx` — Pasa `tenantSlug` al Sidebar
- **Detalle:**
  - Problema: Desde `nexus.grixi.ai/admin/organizations` se podía acceder y controlar toda la plataforma
  - Solución: 6 capas de seguridad implementadas (ver abajo)
  - Guard en TODOS los `loader()` → redirect a `/dashboard` si no es tenant `grixi`
  - Guard en TODOS los `action()` → 403 Forbidden si no es tenant `grixi`
  - Sidebar oculta sección PLATAFORMA si `tenantSlug !== 'grixi'`
- **Decisiones técnicas:**
  - `tenantSlug` viene del hostname (Cloudflare Worker) → no manipulable por el cliente
  - Guard centralizado en `lib/platform-guard.ts` para consistencia
  - Se valida tanto lectura (loaders) como escritura (actions) para evitar POST directo
- **Capas de seguridad verificadas:**
  1. DNS/Cloudflare — Solo `*.grixi.ai` llega al Worker
  2. Worker Edge — `tenantSlug` del hostname real
  3. Loaders (GET) — `isPlatformTenant()` en 5 loaders admin
  4. Actions (POST) — `isPlatformTenant()` en 3 actions admin
  5. DB `platform_admins` — Verificación de admin en tabla
  6. Sidebar (UI) — Links admin ocultos para no-grixi tenants

#### [FEAT] Mejora 3: Rate Limiting — Protección anti-brute-force
- **Archivos afectados:**
  - `wrangler.jsonc` — Binding `ADMIN_RATE_LIMITER` (30 req/60s)
  - `workers/app.ts` — Rate check antes de procesar rutas `/admin`
  - `worker-configuration.d.ts` — Tipo `ADMIN_RATE_LIMITER` en `Env`
- **Detalle:**
  - Usa Cloudflare Workers Rate Limiting nativo (GA Sep 2025)
  - Key: `admin_{CF-Connecting-IP}` — límite por IP
  - 30 requests/minuto → after limit: HTTP 429 + `Retry-After: 60`
  - Binding opcional (`env.ADMIN_RATE_LIMITER?`) para compatibilidad local
- **Decisiones técnicas:**
  - Rate limit solo en `/admin` para no afectar rutas normales
  - JSON response con header `Retry-After` para clientes bien portados

## Sesión 4 — 22:05

### [FIX] Resolución de "Unexpected Server Error" en producción

**Diagnóstico:**
- `grixi.grixi.ai` mostraba "Unexpected Server Error" tras login — el error ocurría ANTES del ErrorBoundary
- Causa raíz: `framer-motion` (692KB) se empaquetaba en el server bundle de Cloudflare Workers
- `framer-motion` accede a `window`/`document` durante evaluación del módulo → crash en Workers (no hay DOM)
- Import estático de `GrixiOrb` en `authenticated.tsx` arrastraba toda la dependencia al SSR

**Acciones:**
- `[FIX]` `app/routes/authenticated.tsx`: Reemplazado import estático de `GrixiOrb` por patrón **client-only** (`useState` + `useEffect` + `import()` dinámico)
- `[REFACTOR]` `app/root.tsx`: ErrorBoundary restaurado a versión producción (sin debug dump, con logging a Workers console)
- Resultado del build: `grixi-orb` separado como chunk independiente (692KB), server bundle reducido de ~2MB a 1.3MB
- 2 deploys exitosos a `grixi.grixi.ai`

**Archivos afectados:**
- `app/routes/authenticated.tsx` — ClientOnlyOrb wrapper
- `app/root.tsx` — ErrorBoundary limpio

### ✅ Verificación
- `pnpm build` exitoso: server 897ms
- `curl -sI grixi.grixi.ai/dashboard` → 302 redirect (sin sesión, comportamiento esperado)
- `curl grixi.grixi.ai/` → HTML válido con loader data completa (traducciones, theme, env vars)

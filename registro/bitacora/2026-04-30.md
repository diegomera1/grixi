### [SECURITY] Hardening del Admin Portal (admin.grixi.ai)

#### [FIX] Bug de Logout no funciona
- **Archivo:** `routes/auth.signout.tsx`
- **Problema:** El sidebar envía `<Form method="post">` pero la ruta solo tenía un `loader` (GET). El POST no era procesado.
- **Solución:** Agregar `action` handler para POST + mantener `loader` como fallback GET
- `scope: "global"` → cerrar todas las sesiones del usuario
- Integrado cleanup de admin session KV en el signout

#### [FEAT] Cookies Session-Only para Admin Portal
- **Archivo:** `lib/supabase/client.server.ts`
- Detección automática de `admin.grixi.ai` via header `host`
- Cookies en admin: sin `maxAge` (session cookie) → cerrar browser = logout
- `SameSite=Strict` + `Secure=true` para anti-CSRF
- Tenants normales mantienen cookies persistentes (Lax)

- Funciones: `requireAdminSession()`, `startAdminSession()`, `clearAdminSession()`

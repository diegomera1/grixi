### [FEAT] Feature Flags — Sistema de control por tenant
- **Migración DB:** `create_feature_flags`
  - Tabla `feature_flags` — definición global de flags (key, name, default_enabled, category)
  - Tabla `feature_flag_overrides` — overrides por organización
  - RLS habilitado, solo lectura para authenticated
  - 17 flags seed para módulos y features existentes
  - Categorías: `module`, `beta`, `experimental`, `general`
- **Server:** `lib/feature-flags.server.ts`
  - `getFeatureFlags(env, orgId)` — merge global defaults + org overrides
  - `isFeatureEnabled(flags, key)` — check individual
  - `requireFeatureFlag(flags, key)` — guard con redirect
  - KV cache con 30s TTL
- **Client:** `lib/hooks/use-feature-flag.ts`
  - `useFeatureFlag("key")` → boolean
  - `useFeatureFlags()` → all flags
  - `<FeatureFlagGate flag="key">` — component guard
- **Integración:** Feature flags en TenantContext y authenticated.tsx loader
- **Archivos:** 3 nuevos, 1 modificado

### [FEAT] Error Tracking — Captura automática de errores en prod
- **Migración DB:** `create_error_logs`
  - Tabla `error_logs` — message, stack, source, level, fingerprint
  - Función `generate_error_fingerprint()` para agrupar errores similares
  - 5 índices optimizados (created_at, fingerprint, source, org, unresolved)
- **API:** `routes/api.errors.ts` — POST /api/errors
  - Recibe errores client/server, parsea browser/OS del UA
  - Genera fingerprint via PostgreSQL function
- **Client:** `lib/error-tracking.ts`
  - `initErrorTracking()` — window.onerror + unhandledrejection
  - `reportError()` — para errores capturados en try/catch
  - Debounce 30s por error único, skip en localhost
  - Fire-and-forget (no bloquea UI)
- **Integración:** Auto-init en authenticated.tsx con user/org context
- **Archivos:** 3 nuevos, 1 modificado

### [FEAT] Analytics de Producto — Tracking de uso
- **Migración DB:** `create_analytics_events`
  - Tabla `analytics_events` — event_name, category, properties, session_id
  - Vista materializada `analytics_daily_summary` para agregados diarios
  - 6 índices (created_at, event_name, org, user, session, category)
- **API:** `routes/api.analytics.ts` — POST /api/analytics
  - Acepta batch de hasta 50 eventos
  - Validación de longitud y sanitización
- **Client:** `lib/analytics.ts`
  - `track(eventName, properties)` — evento genérico
  - `trackPageView()` — auto-detect path y módulo
  - `trackFeature(feature, action)` — uso de features
  - `trackAction(action)` — acciones de usuario
  - Batching: acumula 5s o 10 eventos antes de enviar
  - `navigator.sendBeacon` para envío confiable en page close
  - Session ID persistente en sessionStorage
- **Integración:** Auto page_view en authenticated.tsx
- **Archivos:** 3 nuevos, 1 modificado

### Decisiones técnicas:
1. **Sin dependencias externas:** Analytics, Error Tracking y Feature Flags son self-hosted en Supabase. Cero vendor lock-in, cero costos adicionales.
2. **KV Cache para Feature Flags:** 30s TTL evita queries repetitivos sin sacrificar freshness.
3. **Fingerprint para errores:** PostgreSQL SHA-256 hash agrupa errores similares por message+source+route.
4. **sendBeacon para analytics:** Garantiza envío incluso cuando el usuario cierra la pestaña.
5. **Dynamic import:** Error tracking y analytics se cargan via `import()` dinámico para no bloquear el render inicial.
